# 管理员用户管理设计

## 目标

补齐总设计中的“管理员查看和管理用户基础信息”第一版能力。管理员可以在本地平台中进入用户管理页，查看系统中的账号、用户名、显示名称和角色；普通用户不能访问该页面。

## 范围

- 后端新增用户列表接口：`GET /users`。
- 用户列表返回非敏感字段：`id`、`username`、`displayName`、`role`。
- 前端新增用户管理页 `/users`，仅管理员可访问。
- 管理员导航展示“用户管理”，普通用户导航不展示。
- 修复控制台页面已有中文乱码文案。

## 暂不包含

- 修改用户角色。
- 禁用/删除账号。
- 重置密码。
- 分页和复杂筛选。

这些属于后续管理后台增强，本切片先打通本地可用的基础查看能力。

## 数据流

1. 管理员登录。
2. 进入 `/users`。
3. 前端调用 `GET /api/users`。
4. 后端从 `UserRepository` 查询全部用户。
5. 页面展示用户名、昵称和角色中文标签。

## 验证

- 后端测试证明用户仓库和控制器可以列出用户且不返回密码。
- 前端 API 测试覆盖 `listUsers()`。
- 路由测试证明普通用户不能进入 `/users`，管理员可以进入。
- 页面测试证明用户管理页渲染真实接口数据。
