# 反馈采纳修订题库设计

## 目标

补齐题目反馈闭环：普通用户提交题目答案或解析错误反馈后，管理员在反馈审核页采纳反馈时，可以填写新的标准答案和新的解析。后端采纳反馈后立即更新正式题库中的对应题目，并继续保留修订记录。

## 范围

- 保留现有反馈提交、驳回、待复核流程。
- `POST /questions/feedback/{feedbackId}/accept` 增加可选字段 `correctedAnswer` 和 `correctedAnalysis`。
- 当新答案或新解析为空时，保留题目原值。
- 修订记录继续记录管理员、反馈、题目、修订说明和审核备注。
- 前端反馈审核页展示干净中文，并提供新答案、新解析输入框。

## 数据流

1. 用户提交反馈，反馈进入 `PENDING`。
2. 管理员选择反馈，填写修订说明、审核备注、可选的新答案和新解析。
3. 前端调用采纳接口并提交修订字段。
4. 后端查找反馈关联的题目，合并新答案/解析，更新正式题库。
5. 后端将反馈状态改为 `ACCEPTED` 并返回修订记录。

## 错误处理

- 未选择反馈时，前端提示先选择待处理反馈。
- 后端找不到题目时，返回现有异常路径，避免静默采纳。
- 管理员 ID、修订说明和审核备注仍需要有效内容。

## 验证

- 后端测试证明采纳反馈会更新题目答案和解析。
- 前端测试证明反馈审核页提交 `correctedAnswer` 和 `correctedAnalysis`。
- 运行前端目标测试、后端 question-service 测试、全量前端测试、前端构建、后端测试。
