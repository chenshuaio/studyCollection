# 上传资料生成题目预览设计

**目标：** 用户上传 Java 学习资料后，系统先分析生成题目并展示预览，用户确认后再提交管理员审核入库。

## 范围

- 上传资料支持文本类文件、PDF、DOCX，并补齐 XLSX 资料解析。
- 上传后生成的题目展示在“生成题库”表格中，包含题干、题型、知识点、答案。
- 生成结果不会自动入库，必须点击“提交生成题审核”后才进入管理员审核队列。
- 资料生成规则增强到常见 Java 知识点，避免大多数资料只生成一题概括题。

## 设计

后端沿用 `POST /imports/knowledge/upload`，上传文件先由 `KnowledgeFileTextExtractor` 提取文本，再交给 `KnowledgeQuestionGenerator` 生成题目。`KnowledgeFileTextExtractor` 增加 XLSX 解析，把工作簿文本拼接为可分析内容。生成器继续使用本地规则，补充 JVM、异常、接口/抽象类、集合等常见 Java 规则。

前端导入页保留当前上传控件。上传成功后，将返回的题目放入 `generatedQuestions`，页面表格就是预览区；状态文案明确“请预览后提交审核”。提交按钮只提交当前预览列表。

## 验证

- 后端测试覆盖 `.xlsx` 上传生成题目。
- 生成器测试覆盖多知识点资料可生成多道题。
- 前端测试覆盖上传文件后调用上传接口、展示预览、再提交审核。
- 运行后端 import-service 测试、前端测试和构建。
