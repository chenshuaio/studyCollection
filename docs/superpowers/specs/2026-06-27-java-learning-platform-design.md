# Java 学习题库平台设计

## 背景与目标

本项目是一个 Java 学习题汇总与训练平台。目标是覆盖从入门到精通的 Java 题目，支持客观题、主观题和编程题，允许用户通过多种文件格式导入题库，并能基于题库自动出题、记录错题、生成解析和学习报告。

第一阶段以本地学习使用为主，但整体形态按带账号体系的在线平台设计。系统需要支持普通用户和管理员两类角色，并为后续在线部署、AI 分析、自动判题和服务拆分保留扩展空间。

## 技术栈

- 前端：Vue
- 后端：Java、Spring Boot、MyBatis-Plus
- 数据库：MySQL
- 鉴权：JWT
- 文件存储：第一阶段使用本地文件目录，后续可替换为对象存储
- AI：提供可选在线模型模式；关闭时使用离线规则分析

## 架构设计

系统采用微服务预备版。第一阶段本地运行多个 Spring Boot 服务，服务之间通过 HTTP 调用。后续可以引入注册中心、消息队列、Redis、对象存储和容器化部署。

### 服务划分

- `web-app`：Vue 前端应用，包含用户端和管理员端。
- `gateway-service`：统一入口，负责 JWT 校验、权限拦截、请求路由和统一错误格式。
- `user-service`：负责用户、角色、登录注册、用户资料和管理员权限。
- `question-service`：负责题库、题目、题型、知识点、难度、公共题库、个人题库、题目问题反馈和题目修订历史。
- `import-service`：负责文件上传、格式解析、候选题目生成、导入预览和确认入库。
- `exam-service`：负责练习模式、考试模式、自动组卷、答题会话、客观题评分和考试计时。
- `mistake-service`：负责错题收集、错题重练、掌握状态和薄弱知识点关联。
- `report-service`：负责成绩统计、正确率趋势、薄弱点分析、学习报告和强化题推荐。
- `ai-service`：负责在线 AI 解析、学习建议和薄弱点强化方案生成；当在线模式关闭或调用失败时，回退到规则分析。

## 角色与权限

### 普通用户

- 注册和登录
- 管理个人题库
- 上传并导入题目文件
- 使用公共题库和个人题库生成练习
- 自由组合题目生成个人考试卷
- 参加模拟考试
- 查看错题本
- 反馈题目答案、解析、题干、选项、知识点或难度问题
- 查看学习报告和薄弱点建议

### 管理员

- 管理公共题库
- 审核和维护题目
- 处理题目问题反馈并修订题库
- 管理知识点分类
- 配置考试规则
- 查看和管理用户基础信息

## 页面与用户流程

### 登录页

登录页采用明亮、专业、可信的学习平台风格。

- 使用精致登录卡片、清晰层级和轻微阴影。
- 展示平台核心价值：Java 题库、练习、考试、错题和报告。
- 支持账号密码登录。
- 预留注册入口。
- 登录后根据角色进入用户端或管理员端。

### 用户端

- 学习首页：展示学习概览、最近练习、最近考试、薄弱知识点和推荐强化训练。
- 题库页面：查看我的题库和公共题库，按题型、难度、知识点筛选题目。
- 文件导入：上传文件、查看解析预览、修正字段、确认入库。
- 练习中心：选择知识点、难度、题型和数量，生成练习。
- 考试中心：选择管理员配置的模拟考试，按时间限制答题。
- 答题页：支持客观题、主观题和编程题作答。
- 错题本：按知识点、题型、时间和掌握状态筛选错题，支持重新练习，也可以针对疑似错误题目提交反馈。
- 题目反馈：用户可在答题页、解析页、错题本和题目详情页提交题目问题反馈。
- 报告页：展示得分、正确率、薄弱点、学习趋势、学习建议和强化题推荐。

### 管理员端

- 公共题库管理
- 题目审核与维护
- 题目反馈审核
- 知识点分类管理
- 考试规则管理
- 用户基础管理

## 题目问题反馈与修订

用户发现题目答案或内容存在问题时，可以提交题目反馈。反馈类型包括标准答案错误、解析错误、题干有误、选项有误、知识点或难度标注错误，以及其他问题。

反馈提交后进入管理员端的题目反馈审核队列。管理员审核时可以查看原题、当前标准答案、解析、用户反馈内容、用户提交答案和题目来源。管理员可执行以下操作：

- 采纳反馈并修改题目
- 驳回反馈并填写原因
- 标记为待复核
- 合并重复反馈

当管理员修改公共题库题目时，系统记录题目修订历史，包括修改前内容、修改后内容、修改人、修改时间和关联反馈。后续新练习和新考试使用新版本题目；已经完成的历史答题记录保留当时题目版本，避免旧报告被后续修订影响。

如果某题被确认存在标准答案错误，报告服务可以在后续统计中标记该题为已修订题目，并在重新生成报告时选择排除该题或使用修订后的结果。

## 题型与批改

第一版支持以下题型：

- 单选题
- 多选题
- 判断题
- 填空题
- 简答题
- 编程题

客观题自动判分。简答题和编程题第一版保存用户答案，并展示参考答案、题目解析和关联知识点，不做自动判分。

编程题数据结构预留自动判题字段，包括测试用例、运行状态、运行结果、错误信息、耗时和内存占用。第一版不执行用户代码，后续可接入独立判题沙箱。

## 自动出题

系统支持练习模式和考试模式。

### 练习模式

普通用户选择知识点、难度、题型和题目数量，系统从可用题库中生成练习。可用题库包括公共题库和用户个人题库。

### 考试模式

管理员配置考试规则，包括考试名称、题型比例、知识点范围、难度分布、题目数量和时间限制。用户进入考试后按固定规则答题，提交后生成成绩和分析报告。

普通用户也可以创建个人考试卷。用户可通过知识点、难度、题型和题库范围筛选题目，也可以手动勾选具体题目，组合成一套自定义考试卷。个人考试卷支持设置名称、时间限制和题目顺序，生成后按考试流程作答，并进入错题与报告分析。

## 文件导入

第一版支持以下格式：

- `.json`
- `.csv`
- `.xlsx`
- `.txt`
- `.md`
- `.pdf`
- `.docx`

导入流程为：上传文件，解析为候选题目，展示预览，用户修正题型、答案、解析、知识点和难度，确认后写入题库。

PDF 和 DOCX 解析结果必须经过预览确认后才能入库，避免解析错误污染题库。解析失败时，系统返回失败原因和可修正建议。

## AI 与报告分析

系统提供分析模式开关。

### 在线 AI 模式

开启在线模式时，`report-service` 会调用 `ai-service`，由在线模型生成题目解析、学习建议和薄弱点强化方案。

### 离线规则模式

关闭在线模式，或在线 AI 调用失败时，系统使用规则分析。规则分析基于知识点正确率、错题频率、题型表现、练习趋势和考试成绩生成报告。

### 回退策略

AI 调用失败不会阻断报告生成。系统记录失败原因，并自动回退到离线规则分析。

## 数据流

### 文件导入

用户上传文件到 `import-service`，解析服务生成候选题目并返回前端预览。用户确认后，`import-service` 调用 `question-service` 写入题库。

### 练习与考试

用户在前端选择练习条件或考试，`exam-service` 根据规则向 `question-service` 查询题目并创建答题会话。用户提交答案后，`exam-service` 保存答题记录并完成客观题评分。

### 错题与报告

答题结果进入 `mistake-service` 和 `report-service`。错题服务维护错题和掌握状态。报告服务汇总答题表现，并根据配置选择 AI 分析或规则分析。

### 题目反馈与修订

用户从答题页、解析页、错题本或题目详情页提交反馈到 `question-service`。管理员在题目反馈审核页处理反馈。若反馈被采纳，`question-service` 写入题目新版本和修订历史，并将反馈状态更新为已采纳。报告服务在统计时识别已修订题目，避免错误题目继续影响学习分析。

## 错误处理

- 文件解析失败：返回失败原因和可修正建议。
- 题目字段不完整：进入待修正状态，不直接入库。
- 题目反馈内容为空：拒绝提交并提示用户填写具体问题。
- 重复反馈：允许提交，但管理员端按题目和问题类型聚合展示。
- AI 调用失败：自动回退离线规则分析。
- 考试中断：保存已答进度，允许恢复或重新进入。
- 权限不足：网关返回统一无权限提示。
- 服务不可用：网关返回统一错误格式，并保留错误追踪信息。

## 第一版范围

第一版需要打通完整学习闭环：登录、题库导入、练习、考试、用户自定义组卷、答题、错题、题目问题反馈、管理员修订题库、报告和 AI/规则双模式分析。

第一版不包含真正的 Java 代码自动判题，不包含在线支付、班级管理、教师角色、复杂运营后台和生产级容器编排。
