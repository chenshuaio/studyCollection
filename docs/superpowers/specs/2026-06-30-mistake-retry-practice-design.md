# 错题重练设计

## 目标

让错题本中的“重新练习”真正进入指定错题，而不是只跳转到普通练习页。用户从错题本选择某一道错题后，练习中心优先展示该题，提交后继续使用现有评分、错题记录和反馈流程。

## 范围

- 前端本地实现，不新增后端接口。
- 错题本点击“重新练习”时，将错题 `questionId` 和 `questionTitle` 写入 `sessionStorage`。
- 练习中心加载题库后，优先把对应题目排到当前练习第一题。
- 如果题目已被管理员删除或搜索不到，练习中心回退普通练习并提示未找到指定错题。

## 数据流

1. 用户进入错题本。
2. 点击某条错题的“重新练习”。
3. 前端保存重练目标并进入 `/practice`。
4. 练习页加载正式题库，匹配 `questionId`。
5. 匹配成功时显示“错题重练”，并优先展示该题。

## 验证

- 错题页测试覆盖点击重练会写入本地重练目标。
- 练习页测试覆盖有重练目标时优先展示对应题目。
- 保持现有练习、错题状态更新测试通过。
