# 真实题库驱动练习与考试设计

## 目标

让练习中心和考试中心使用正式题库中的题目，而不是前端硬编码样例。管理员审核通过的导入题进入正式题库后，普通用户应能在练习和自定义考试中使用这些题目。

## 范围

本轮实现第一阶段闭环：

- 练习中心从 `/questions` 拉取正式题库，支持按题干关键字、知识点、难度、题型和数量生成练习。
- 考试中心从 `/questions` 拉取正式题库，用户可勾选真实题目生成个人考试卷。
- 答题页使用题目快照，提交时携带用户答案、标准答案和解析，后端完成判分并累计用户做题统计。
- 因当前题目模型没有选项字段，真实题库题目先使用文本输入答案；原硬编码样例不再作为主要题源。

## 数据流

1. 管理员审核通过导入题后，题目写入正式题库。
2. 练习中心/考试中心调用题库搜索接口读取正式题目。
3. 用户答题提交到 `practice-service`，请求包含 `questionId`、用户答案、标准答案和解析。
4. 后端优先使用题目快照判分；如果没有快照则兼容原本样例题库。
5. 错题仍进入错题本，控制台统计继续使用用户做题统计和错题数据。

## 非本轮范围

- 不新增数据库选项表。
- 不实现编程题自动判题。
- 不实现复杂考试规则配置。
- 不实现历史题目版本快照存储。
