# 响应式布局设计

**目标：** 前端页面在电脑端和手机端都可用，电脑端保留现有高效布局，手机端避免横向溢出和按钮挤压。

## 范围

- 保留桌面端侧边栏、双栏和三栏布局。
- 在手机端将主壳层切成单列，侧边导航变为顶部横向滚动导航。
- 让 header 操作区、账户信息、按钮组在窄屏下自动换行。
- 让表格区域在手机端横向滚动，而不是撑宽整个页面。
- 不改后端接口和业务流程。

## 设计

全局样式仍集中在 `frontend/src/styles/theme.css`。新增桌面默认规则和两个断点：`max-width: 860px` 用于平板和窄屏基础重排，`max-width: 560px` 用于手机端密度优化。页面组件不新增复杂移动端状态，避免引入抽屉菜单和额外交互。

表格使用现有 `.table-panel` 作为滚动容器，在小屏下设置 `overflow-x: auto` 和表格最小宽度。导航使用横向滚动，保持所有入口可访问。按钮、表单、文件上传和操作区在手机端宽度占满，降低误触和文字溢出风险。

## 验证

- 增加前端测试，确认核心页面仍渲染全局响应式结构。
- 运行 `npm test` 和 `npm run build`。
- 使用浏览器检查桌面宽屏和手机视口，确认无全页横向溢出，导航、表格、按钮可访问。
